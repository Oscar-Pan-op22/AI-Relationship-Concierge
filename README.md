# 同频 Phase 1 工作台

这是同频 `Phase 1` 的一个零依赖本地实现：

`Twin 建档 -> 候选人录入 -> 结构化尽调报告 -> 提问草稿包`

## 已实现

- `Twin 建档`
  - 关系目标
  - 偏好城市
  - 必须满足项与硬性雷区
  - 沟通风格、结婚时间预期、孩子态度
  - 家庭边界与财务观
  - 敏感议题授权
- `候选人录入`
  - 基础资料字段
  - 自由文本资料、聊天摘要、用户备注
- `筛选引擎`
  - 结构化事实提取
  - 匹配度矩阵
  - 风险信号识别
  - 信息缺口清单
  - 下一步建议
  - 温和提问草稿
- `本地持久化`
  - 生成结果存储在 `data/screenings.json`

## 当前未实现

- 与候选人的实时对话
- 自动外发消息
- 真实 LLM 或外部 API 接入
- 完整相亲网络、匹配市场或信任体系

当前分析引擎是启发式、可替换的模块，后续可以直接切换成真实 Agent 或模型 API。

## 启动

```bash
npm.cmd start
```

打开 `http://localhost:3000`

## 测试

```bash
npm.cmd test
```

## 结构

- `src/server.js`：HTTP 服务与 API 路由
- `src/lib/screeningEngine.js`：Phase 1 分析流水线
- `src/lib/storage.js`：本地报告持久化
- `public/`：单页工作台界面
- `test/`：筛选引擎测试
